# Contribution à Obsidia

Merci de votre intérêt pour Obsidia ! Nous accueillons les contributions qui renforcent la sécurité et la transparence de la gouvernance déterministe.

## Comment contribuer ?

1.  **Signaler des bugs :** Utilisez les GitHub Issues pour signaler tout comportement inattendu ou toute faille de sécurité potentielle.
2.  **Proposer des fonctionnalités :** Ouvrez une discussion ou une issue pour discuter de nouvelles idées.
3.  **Soumettre des Pull Requests :**
    *   Clonez le dépôt.
    *   Créez une branche pour votre fonctionnalité (`git checkout -b feature/ma-feature`).
    *   Assurez-vous que vos changements ne violent aucun invariant critique (vérifiez avec Lean 4 si possible).
    *   Soumettez votre PR avec une description détaillée.

## Normes de Code
*   **Python :** Suivez la PEP 8 pour les scripts du moteur Sigma.
*   **Lean 4 :** Documentez chaque théorème et chaque preuve formelle.
*   **React :** Utilisez des composants fonctionnels et TypeScript pour une sécurité maximale du type.

---
*Obsidia est un projet communautaire dédié à la sécurité par la preuve.*
