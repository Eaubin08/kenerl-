# Glossaire Obsidia

Ce glossaire définit les termes techniques utilisés dans l'application Obsidia et son dépôt GitHub.

## Termes Clés

### 1. Gouvernance Déterministe
Une approche de sécurité où chaque action est validée par des règles mathématiques strictes avant d'être exécutée. Contrairement aux systèmes probabilistes (IA classique), le résultat est prévisible et garanti.

### 2. Sigma Engine (Moteur Sigma)
Le module de surveillance de la stabilité d'Obsidia. Il mesure la "vibration" ou le "bruit" des transactions. Si l'instabilité dépasse un certain seuil, le système se verrouille automatiquement.

### 3. X-108 (Protocole)
Le protocole de freinage temporel. Il impose un délai de 108 secondes sur les actions critiques pour permettre une intervention humaine ou une validation supplémentaire, neutralisant ainsi les attaques automatisées ultra-rapides.

### 4. Preuve de Merkle (Merkle Proof)
Une structure cryptographique qui permet de prouver l'intégrité d'un ensemble de données. Si un seul bit est modifié, la racine de Merkle change, alertant immédiatement le système d'une corruption.

### 5. Lean 4
Un langage de programmation et un assistant de preuve utilisé pour vérifier mathématiquement que le code source d'Obsidia respecte ses spécifications de sécurité. C'est le niveau de confiance le plus élevé possible en informatique.

### 6. Invariant Critique
Une règle de sécurité qui ne doit jamais être violée (ex: "Le solde d'un portefeuille ne peut pas diminuer de plus de 10% en une seule transaction"). Obsidia surveille ces invariants en permanence.

### 7. Preuve de Patience
Le concept selon lequel le temps est une barrière de sécurité. En forçant un attaquant à attendre, on lui retire son avantage principal : la vitesse.
